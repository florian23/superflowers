Problem: The processOrder function in src/complex.js had a cyclomatic complexity of ~7,
exceeding the architecture.md constraint of "Cyclomatic < 5 per function".

Root cause: Deeply nested if-else statements (5 levels deep) handling express shipping
logic with multiple conditions (type, weight, destination, insurance, fragile).

Solution: Decomposed the single function into three smaller functions using Extract Method refactoring:

1. processOrder (complexity: 3) — top-level dispatch: express vs standard/freight
2. getExpressShipping (complexity: 3) — handles weight threshold and destination routing
3. getHeavyInternationalExpress (complexity: 4) — handles insurance + fragile combinations

All three functions now have cyclomatic complexity below 5.
Functional behavior is fully preserved (all 7 test cases pass).

Cyclomatic complexity breakdown:
  - getHeavyInternationalExpress: base(1) + if-insurance(1) + &&-fragile(1) + ternary(1) = 4
  - getExpressShipping: base(1) + if-weight(1) + if-destination(1) = 3
  - processOrder: base(1) + if-type(1) + ternary-weight(1) = 3
