# Security Constraints

Für dieses Projekt gelten folgende Security-Richtlinien aus dem Unternehmens-Constraint-Repo:

## Aktiv

- **SEC-001**: Encryption at Rest — Alle Daten verschlüsselt speichern
- **SEC-002**: API Authentication — OAuth 2.0 / JWT für alle Endpunkte

## Nicht relevant

- **SEC-003**: Network Segmentation — wird vom Infra-Team separat gehandhabt, nicht Aufgabe der Entwickler
